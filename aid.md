``` git init


Разгитить папку 


``` rm -rf .git

ИНФОРМАЦИЯ О ХЭШЕ


```mermaid

Graph LR;

* Git преобразует информацию о коммитах с помощью алгоритма SHA-1 и для каждого из них расчитывает уникальный индентификатор - хеш

* Хеш - основной идентификатор коммита и позволяет узнать его автора, дату и содержимое закоммиченных файлов.

* Все хеши, а также таблицу соответствий %% хеш информация о коммите Git хранит в папке .git


ИНФОРМАЦИЯ О ЛОГАХ И НАДПИСЬ HEAD

* Можно вызвать не только полный лог, но и сокращённый - это делается командой 

```git log --oneline

* В сокращённом логе выводятся сокращённые хеши - их можно использовать точно так же, как и полные.

-------------------------------------------------------------

* В числе прочих файлов в папке .git есть служебный фаил HEAD. Он указывает на самый свежий коммит.

* Вместо хеша последнего коммита можно написать слово HEAD- Git вас поймёт.

Комманды для отката коммитов,файлов до предыдущей версии

Команда git restore --staged <file> переведёт файл из staged 
обратно в modified или untracked.

Команда git reset --hard <commit hash> «откатит» историю до коммита с хешем <hash>. Более поздние коммиты потеряются!


Команда git restore <file> «откатит» изменения в файле до последней сохранённой (в коммите или в staging) версии.
